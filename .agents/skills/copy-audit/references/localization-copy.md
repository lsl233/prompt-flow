# Localization Copy Reference

Use this file when reviewing multilingual content quality, terminology consistency, and localized UX text.

## Core Rules

- Localize all user-facing strings that belong to the localized experience.
- Keep terminology consistent across pages, components, and metadata.
- Preserve brand, legal, and regulated terms according to source-of-truth guidance.
- Avoid mixed-language UI unless required by product design.

## What To Check

- page title and meta description
- headings and body copy
- CTA labels and system messages
- form labels, errors, helper text
- image alt text
- nav, breadcrumbs, tabs, and badges

## Red Flags

- untranslated placeholders or keys rendered in UI
- one locale reusing another locale's title or description
- terminology drift across translation files
- machine-translated copy that changes the product meaning
- inconsistent tone between pages in the same locale

## Locale-Specific UX

Check for localization of:

- date and time wording
- currency wording
- number formatting labels
- regional spelling variants
- market-specific shipping, tax, or legal notes

These affect trust and comprehension even when the raw feature works.
