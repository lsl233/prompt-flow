# Product and UX Copy Reference

Use this file when reviewing clarity, consistency, conversion, task completion, and interaction-state wording.

## Product Clarity

Check:

- the user can tell what the page or feature does within seconds
- core nouns are stable across the product
- actions describe outcomes, not internal implementation
- pricing, limits, or availability language is unambiguous when present

Flag:

- inconsistent names for the same concept
- internal team vocabulary exposed to users
- CTA labels that hide what happens next

## CTA Labels

Prefer labels that state the outcome:

- "Create project"
- "Start free trial"
- "Download invoice"

Be cautious with:

- "Submit"
- "Continue"
- "Next"
- "Confirm"

These may be acceptable only when the surrounding context is unmistakable.

## Forms

Check:

- labels are explicit
- helper text answers foreseeable questions
- validation errors explain both the issue and the fix
- success states confirm the result

Flag:

- placeholder-only inputs
- errors such as "Invalid input" with no guidance
- destructive actions with weak confirmation copy

## Empty, Loading, and Error States

Check:

- empty states explain why nothing is shown
- empty states offer a sensible next action
- loading states set expectation when waits are meaningful
- error states say what failed, what the user can do now, and whether data is safe

Flag:

- "No data" with no next step
- passive errors that shift blame to the user without explanation
- success toasts that are too vague to confirm the action

## Tone and Consistency

Check:

- tone matches the product and the moment
- onboarding is encouraging without being vague
- destructive moments are explicit and calm
- support/help text is direct and not patronizing

Flag:

- playful tone in high-risk flows
- abrupt robotic tone in guided experiences
- inconsistent punctuation, capitalization, and terminology in shared UI
